## 🚀 Как запустить

### Вариант 1: Просто открыть
Откройте файл `index.html` в браузере.

### Вариант 2: GitHub Pages
1. Загрузите проект на GitHub
2. Settings → Pages → Source: `main` branch
3. Сайт будет доступен по адресу `https://ваш-username.github.io/Plato-Pronto`

## 🔐 Настройка безопасности (ОБЯЗАТЕЛЬНО!)

### ⚠️ ВНИМАНИЕ: Старый Telegram-токен скомпрометирован!
Если ваш токен когда-либо был в публичном репозитории — **немедленно создайте новый!**

---

### Шаг 1: Создайте новый Telegram-бот

1. Откройте Telegram и найдите **@BotFather**
2. Отправьте команду `/mybots`
3. Выберите вашего бота
4. Отправьте `/revoke` — это **удалит старый токен**
5. Отправьте `/token` — скопируйте **новый токен**
6. ⚠️ **Никогда не публикуйте этот токен!**

### Шаг 2: Настройте Google Apps Script (бесплатный серверless-прокси)

1. Откройте [script.google.com](https://script.google.com) (войдите в Google аккаунт)
2. Нажмите **"Новый проект"**
3. Удалите весь код в редакторе
4. Скопируйте содержимое файла `backend/google-apps-script.js` и вставьте в редактор
5. Замените значения в блоке `CONFIG`:
   - `TELEGRAM_BOT_TOKEN` — вставьте ваш **новый токен** от @BotFather
   - `TELEGRAM_CHAT_ID` — ваш Chat ID (можно узнать у [@userinfobot](https://t.me/userinfobot))
   - `API_KEY` — придумайте **сложную строку** (минимум 32 символа, например: `aB3x9ZpL2mN8qR5wK7vT4yC6jF1hD0sE`)
6. Нажмите **"Развернуть"** → **"Новое развертывание"**
7. Тип: **"Веб-приложение"**
8. Кто имеет доступ: **"Все"**
9. Нажмите **"Развернуть"**
10. **Скопируйте URL** — это ваш `BACKEND_URL`

### Шаг 3: Вставьте URL в сайт

Откройте файл `index.html` и найдите блок `CONFIG` (в начале `<script>`):

```javascript
const CONFIG = {
    BACKEND_URL: 'ВАШ_URL_GOOGLE_APPS_SCRIPT',  // ← Вставьте URL из шага 2
    API_KEY: 'ВАШ_API_КЛЮЧ'                      // ← Вставьте тот же ключ что и в Google Apps Script
};
```

### Шаг 4: Проверьте что всё работает

1. Откройте `index.html` в браузере
2. Добавьте товар в корзину
3. Оформите заказ
4. Проверьте что сообщение пришло в Telegram

## 🔒 Реализованные меры безопасности

| Мера | Описание |
|------|----------|
| 🔑 Скрытый токен | Telegram-токен хранится только в Google Apps Script, не в коде сайта |
| 🛡️ API ключ | Дополнительная аутентификация запросов к прокси |
| 🚫 Защита от XSS | Все пользовательские данные экранируются перед вставкой в HTML |
| ⏱️ Throttle | Не более 1 заказа в 30 секунд |
| 🔒 CSP | Content Security Policy ограничивает загрузку ресурсов |
| 🚫 Referrer | `no-referrer` — сайт не передаёт информацию о себе другим сайтам |
| ✅ Валидация | Проверка имени, телефона и города перед отправкой |
| 🔄 Двойная отправка | Форма блокируется пока запрос не завершится |
